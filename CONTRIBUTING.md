## <a name="english"></a>English

Thank you for your interest in contributing to TPU Programming Tutorials! We welcome contributions from the community.

### ğŸŒŸ How to Contribute

There are many ways to contribute to this project:

1. **Add new tutorials** - Share your knowledge and create new tutorials
2. **Improve documentation** - Enhance existing documentation or fix typos
3. **Add code examples** - Contribute practical code examples
4. **Report issues** - Report bugs or suggest improvements
5. **Translate content** - Help translate tutorials to other languages
6. **Review pull requests** - Help review contributions from others

### ğŸ“ Contribution Guidelines

#### 1. Fork and Clone

```bash
# Fork the repository on GitHub, then clone your fork
git clone https://github.com/YOUR_USERNAME/TPU-Programming-Tutorials.git
cd TPU-Programming-Tutorials

# Add upstream remote
git remote add upstream https://github.com/Zhighway777/TPU-Programming-Tutorials.git
```

#### 2. Create a Branch

```bash
# Create a new branch for your contribution
git checkout -b feature/your-feature-name
```

#### 3. Make Your Changes

- Follow the existing code style and structure
- Add clear comments and documentation
- Test your code examples to ensure they work
- Update relevant documentation

#### 4. Commit Your Changes

```bash
# Stage your changes
git add .

# Commit with a descriptive message
git commit -m "Add: Brief description of your changes"
```

Use clear commit messages:
- `Add:` for new features or content
- `Fix:` for bug fixes
- `Update:` for updates to existing content
- `Docs:` for documentation changes

#### 5. Push and Create a Pull Request

```bash
# Push to your fork
git push origin feature/your-feature-name
```

Then create a Pull Request on GitHub with:
- Clear title describing the change
- Detailed description of what you've added or fixed
- Any relevant issue numbers (e.g., "Fixes #123")

### ğŸ“š Tutorial Guidelines

When creating tutorials, please follow these guidelines:

#### Structure
- Start with clear learning objectives
- Include prerequisites
- Provide step-by-step instructions
- Add code examples with explanations
- Include expected output or results
- End with a summary and next steps

#### Code Examples
- Use clear variable names
- Add comments for complex code
- Test all code examples
- Include error handling where appropriate
- Specify required dependencies and versions

#### Documentation
- Write in clear, concise language
- Use proper markdown formatting
- Include diagrams or images where helpful
- Provide links to related resources

### ğŸ¨ Style Guide

#### Markdown Formatting
- Use consistent heading levels
- Add blank lines between sections
- Use code blocks with language specification
- Add alt text for images

#### Code Style
- Follow PEP 8 for Python code
- Use 4 spaces for indentation
- Keep lines under 80 characters when reasonable
- Add docstrings for functions and classes

### ğŸ› Reporting Issues

When reporting issues, please include:
- Clear description of the problem
- Steps to reproduce the issue
- Expected vs actual behavior
- Environment details (OS, Python version, etc.)
- Error messages or screenshots if applicable

### âœ… Pull Request Checklist

Before submitting a pull request, ensure:
- [ ] Code runs without errors
- [ ] Documentation is updated
- [ ] Commit messages are clear
- [ ] Branch is up to date with main
- [ ] Changes follow the style guide

### ğŸ“§ Questions?

If you have questions, feel free to:
- Open an issue for discussion
- Use GitHub Discussions
- Contact the maintainers

### ğŸ™ Recognition

All contributors will be recognized in our repository. Thank you for helping make TPU programming more accessible!

### ğŸ“Œ Source of Truth

- English documentation is considered the authoritative version
- Translations may be adapted for clarity and pedagogy
- Technical correctness should follow official TPU / XLA behavior
